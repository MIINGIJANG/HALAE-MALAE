## 📝 1. Business purpose
* **프로젝트 배경**: 카카오톡, LMS, 학교 홈페이지 등으로 데드라인 정보가 심각하게 파편화되어 관리 효율이 저하됨.
* **기획 동기**: 기존 일정 앱은 정적인 정보 나열에 그쳐 마감에 대한 심리적 압박감이나 실행 동기를 부여하지 못함.
* **최종 목표**: 파편화된 데드라인을 단일화하고, 'D-Day의 압박감'을 시각적으로 구현한 능동형 멘탈 케어 앱 개발.
* **타겟 마켓**: 기존 일정 앱에 만족하지 못하는 20대 대학생 및 미니멀한 감성을 원하는 사용자.

<br/>

* **할래말래 (Target App)**: 데이터를 바탕으로 D-Day를 계산하고 푸시 알림을 발송하는 핵심 시스템.
* **사용자 (User)**: 일정을 직접 입력하고, 통합 리스트와 알림을 통해 일정 관리를 받는 주체.
* **외부 정보 출처**: 카톡, LMS, 홈페이지 등 사용자가 마감 데이터를 획득하는 외부 환경.
* **기존 일정 앱**: 정적 정보만 제공하여 '할래말래'로 대체되어 사용이 중단되는 대상.

## 📌 3. Use case list

| 유스케이스 명 | Actor | 상세 설명 |
| :--- | :--- | :--- |
| **1) 일정 직접 입력** | 사용자 | 카톡, LMS 등에서 확인한 일정을 앱에 직접 입력하여 등록한다. |
| **2) 통합 리스트 조회** | 사용자 | 마감일(D-Day)이 임박한 우선순위 순으로 정렬된 리스트를 조회한다. |
| **3) 통합 캘린더 조회** | 사용자 | 입력한 모든 일정 데이터를 앱 내 통합 달력 UI 위에서 한눈에 확인한다. |
| **4) 마감 임박 알람** | 사용자 | 설정된 마감 시간이 임박했을 때 푸시 알림(Push Notification)을 수신한다. |
| **5) 일정 완료 처리** | 사용자 | 일정을 완수했을 경우 체크하여 완료 상태로 변경하고 UI에 반영한다. |

| **목표** | 수동 입력 기능 구현 | 리스트 뷰 정렬 구현 | 달력 UI 상호작용 구현 | D-Day 기반 타이머 구현 |

## ⚠️ 5. Problem statement & NFRs
* **기술적 과제**: 수동 입력 피로도 최소화 UI, 백그라운드 알림 정확성 확보, 시각적 압박감과 심미성의 조화.
* **비기능 요구사항(NFRs)**: 
    * **신뢰성**: 알림 설정 시간에 오차 없이 100% 트리거되어야 함.
    * **사용성**: 신규 일정 등록 시 3단계 이내로 완료되는 직관적인 UI 제공.
    * **성능**: 화면 전환 시 지연 없는 즉각적인 렌더링 보장.

<br/>

## 📖 6. Glossary
* **LMS**: 학습관리시스템으로 주요 마감 정보의 출처입니다.
* **파편화**: 일정이 여러 플랫폼에 분산되어 관리가 어려운 상태입니다.
* **D-Day**: 과제 마감일까지 남은 기한으로 행동 유도를 위한 핵심 지표입니다.
* **가변적 UI**: 마감 임박도에 따라 테마나 마스코트가 변화하는 설계 원리입니다.
* **로컬 푸시 알림**: 서버 없이 기기 내부 기능을 활용해 알림을 띄우는 방식입니다.
